Undeleting Items in the Log View

To undelete a deleted item:

  1. Select a folder in the working copy or repository browser which once contained the file or folder you wish to undelete.

    If you are unsure of the item’s location, or the parent folder also no longer exists then select a folder higher up the working copy or repository folder structure.

  2. Switch to the log view by selecting View > Log (key equivalent âŒ˜2) or by clicking the log button in the browser view toolbar:

    Log

  3. Locate the revision when the item was deleted in the Log View. You may need to navigate back through the history using the Load Next button located at the bottom of the log view.

    You can use the Deleted filter in the log view’s filter bar to more easily locate deletions.

  4. Expand the revision's Changes section and select the item to undelete from the Deleted section.

  5. Click the Undelete button in the log view’s toolbar:

    Undelete

  6. An options window is displayed next to the selected item from the Changes section.

    Specify the name and location that the item should be undeleted to. If the Log View is displaying the log for a working copy folder, the options window will suggest undeleting to the working copy.

  7. Click Undelete.

  8. The selected item will be undeleted by being copied to the specified location. The item’s original history will be preserved.
